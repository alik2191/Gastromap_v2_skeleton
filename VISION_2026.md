# GastroMap: Видение и Стратегия 2026

## 1. Текущее состояние (Техническое и Продуктовое)

### Обзор продукта
GastroMap — это современное веб-приложение на React (Vite) для поиска ресторанов, сочетающее в себе интерактивные карты, персональные профили и умного ИИ-ассистента. Проект построен на архитектуре Feature-Sliced Design (FSD), что обеспечивает высокую масштабируемость.

### Ключевые возможности (Реализованные)
*   **ИИ-Ассистент (GastroAIChat):** Полноценный чат-интерфейс для персональных рекомендаций на основе предпочтений пользователя.
*   **Интерактивные карты:** Система на базе `Leaflet`, позволяющая искать заведения в разных городах.
*   **Профили и предпочтения:** Развитая система управления профилем, где пользователь может указать свою "Foodie DNA", атмосферу и предпочтения по кухне.
*   **Управление данными (Admin Panel):** Мощная панель администратора с функциями импорта/экспорта локаций через CSV и сложной фильтрацией.
*   **Адаптивный дизайн:** Полная поддержка мобильных устройств и темная тема.

### Технологический стек
*   **Frontend:** React 18, Vite, **Zustand** (вместо Redux) для легкого управления состоянием.
*   **Стилизация:** Tailwind CSS 3.4 + DaisyUI для быстрой и качественной верстки.
*   **Анимации:** Framer Motion и Lenis для плавного скролла.
*   **Тестирование:** Vitest + React Testing Library (Unit и функциональные тесты).
*   **Состояние данных:** Сейчас проект использует **Mock-сервисы** и локальное хранилище (`persist` в Zustand) для имитации авторизации и работы с данными.

### Технический долг и первоочередные задачи
*   **Backend Integration:** Переход от Mock-данных к реальному бэкенду (Firebase или Supabase).
*   **PWA:** Настройка полноценного офлайн-режима (Vite PWA Plugin сейчас не сконфигурирован).
*   **SEO:** Оптимизация SPA для поисковых систем (SSR или пре-рендеринг).

## 3. Новые "Killer Features" для реализации

### 1. Поиск "Mood & Bio-Sync" (ИИ + Здоровье)
*   **Концепция:** Интеграция с данными о здоровье. Если пользователь прошел 10к шагов — ИИ предлагает места с питательным ланчем.
*   **Реализация:** HealthKit / Google Fit API + контекстные промпты для LLM.

### 2. AR "Dish-o-Vision" (Дополненная реальность)
*   **Концепция:** Визуализация блюд в 3D через камеру смартфона прямо на столе или при наведении на меню.
*   **Реализация:** WebXR.

### 3. "Dine With Me" (Социальный радар)
*   **Концепция:** Временный статус на карте "Готов пообедать вместе" для поиска компании среди друзей или коллег поблизости.

### 4. Модуль бронирования и геймификации (План)
*   **Booking Engine:** Внедрение системы резервирования столиков напрямую через приложение.
*   **Badges & Rewards:** Система достижений за посещение новых мест или написание ИИ-обзоров.

### 5. Кнопка "Анти-решение"
*   **Концепция:** Групповой выбор места через механику свайпов, где алгоритм находит пересечение интересов всех участников.

## 4. Техническая дорожная карта

### Фаза 1: Оживление данных (Short-term)
*   **Интеграция БД:** Подключение реальной базы данных для синхронизации профилей между устройствами.
*   **PWA & Performance:** Добавление Service Workers для работы карт в офлайне и ускорение загрузки через оптимизацию ресурсов.

### Фаза 2: Глубокий ИИ и Поиск (Mid-term)
*   **Векторный поиск:** Использование векторных БД (например, Supabase Vector) для семантического поиска ("где выпить кофе и поработать").
*   **Edge Functions:** Перенос логики ИИ на Edge для мгновенного отклика.

### Фаза 3: Социальная экосистема (Long-term)
*   **Native Apps:** Выпуск мобильных приложений на React Native.
*   **B2B Portal:** Личный кабинет для владельцев ресторанов с аналитикой популярности заведения в ИИ-запросах.
